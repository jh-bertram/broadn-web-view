## Anomalies / notes for the auditor and PM
1. **Determinism bug found + fixed during self-verification**: `coverage_summary.unique_api_calls` was initially wired to `stats["live_calls"]` (this-run's actual network activity — 1130 on the first cold-cache run, 0 on any subsequent warm-cache run), which meant a cache-only rebuild would NOT be byte-identical to the original live-run output even though the underlying data was identical. Root cause: conflating "how many live network calls did THIS invocation make" (a diagnostic, printed to stdout only) with "how many distinct API calls does this dataset require" (`len(unique_keys)`, a deterministic property of the classified dataset). Fixed to `len(unique_keys)`; re-verified across 4 runs as above. This is exactly the class of self-report-vs-actual-execution gap the GATE-RUN auditor step exists to catch — I caught it myself during the mandated re-run self-check, but flagging explicitly since it changed the output after the "first" live run.
2. `TEMP` xlsx column (needed for ST's p14-002 field-temperature cross-check) contains garbled non-numeric tokens on visual inspection (e.g. `'N_A'`, `'m_R'`, `'_Q1'`, `'000'`) rather than plausible temperature readings — flagging for ST's attention; out of my scope to fix or investigate further (per PM plan risk_flags, this was already flagged as "FIELD-TEMP COLUMN UNVERIFIED").
3. `unique_api_calls=1130` vs. the brief's "a few hundred" estimate — see Coverage summary section above; not a defect, just a dataset-scale observation.
4. No secrets/API keys (Open-Meteo requires none); `User-Agent` is descriptive (`BROADN-WebView-CovariateEnrichment/1.0 (+https://github.com/jh-bertram/broadn-web-view; ...)`), no personal contact info embedded.
5. Static-site files (`index.html`, `assets/**`) — confirmed untouched (`git diff --stat` empty for both).
