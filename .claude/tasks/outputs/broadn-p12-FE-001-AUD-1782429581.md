mobile user has no access to STORY sections, slice tools, or the Explorer. This is an
explicit acceptance criterion (B6) and the central promise of the contract (§1: "improves
mobile — everything routes through the existing drawer"). Returns to FE.

## Required fix (returns to FE#1)

**File:** `assets/app.js` — `openMobileDrawer()` (line 4189) and `closeMobileDrawer()` (line 4206).
**Problem:** the `hidden lg:flex` (mobile display:none) is on `#slice-sidebar-wrapper`
(`index.html:88`), not on `#slice-sidebar`. The drawer toggles the wrong element, so the
panel is trapped inside a `display:none` ancestor.
**Required outcome:** when the drawer opens on mobile, the rail (STORY + EXPLORE groups) must
become visible and interactive; selecting any item must close the drawer and return focus to
the trigger (closeMobileDrawer already calls `trigger.focus()`). Re-run B6 live at 390px:
open drawer → rail visible → click a STORY item and the Explorer item → drawer closes, correct
pane shows, focus returns, 0 console errors.
**Suggested approach (FE to finalize):** have openMobileDrawer also remove `hidden` from (and
restore it to) `#slice-sidebar-wrapper`, or relocate the drawer-mode class toggling onto the
wrapper so the fixed panel is no longer descended from a `display:none` node. Verify the
desktop sticky layout (lg:flex) is unaffected.

Everything else (SA, desktop B1–B5/B7–B9, SX) is clean — this is a single, well-scoped fix.
