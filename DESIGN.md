# Design System — BROADN Aerobiome Dashboard
> Version: 2.0.0 | Updated: 2026-06-25 | Generated by: UI#1 (broadn-p10-design-implementation-UI-001)

## Constitution

Non-negotiable rules for this app's visual identity.

- **Light mode primary:** the default and designed experience is light mode on a warm stone base — never use dark backgrounds as the primary surface
- **BROADN teal is the brand anchor:** deep teal `#0c5454` is the institutional accent for nav active states, primary headings, primary buttons, hero banner, and KPI icon halos; bright teal `#0c9cb4` is used for active/accent/links/underlines. **This OVERRIDES v1.0.0's "CSU Green is the brand anchor" rule.** CSU green (the former brand anchor) is retired from the UI entirely — do not use it in any new code or design context. All migration targets are documented in the v1→v2 Migration Table.
- **Brand ≠ Data:** BROADN teal (`#0c5454` / `#0c9cb4`) must NOT appear in the sample-type data palette (`SAMPLE_TYPE_COLORS`). Brand colors and data-encoding colors are kept separate to prevent brand–data color confusion. Chart data series use Okabe-Ito (see § Sample-Type Data Palette).
- **Data clarity over decoration:** chart and table content is the product — no decorative gradients, drop shadows heavier than `shadow-sm`, or animations that compete with data
- **No emoji icons in production UI:** KPI icon halos use proper SVG icons (lucide or equivalent) — emoji are placeholder only
- **Tailwind utility-first:** all layout and spacing via Tailwind classes — no custom CSS except for Chart.js container sizing and skeleton animation, which cannot be expressed as utilities
- **Vanilla stack — no framework imports:** the app is HTML + Tailwind CDN + Chart.js; do not introduce React, Vue, or other component frameworks without a separate architectural decision
- **Minimum touch target: 44×44px** for all interactive elements (nav links, filter buttons, table sort controls)
- **Accessible charts:** every Chart.js canvas must have an `aria-label` and a corresponding accessible data summary (hidden table or `aria-describedby` text)

### Orange Token Consolidation (Four-Orange Collapse)

v1 had four competing orange values in the UI simultaneously. v2 collapses all **filter-signal oranges** to ONE token: `--color-filter-accent = #c2410c`.

| Old Value | Context | v2 Action |
|-----------|---------|-----------|
| `#ea6c00` (`CHART_COLORS.orangeAccent`, bar-highlight) | filter signal on chart bar | → `#c2410c` (`--color-filter-accent`) |

## Sample-Type Data Palette (SAMPLE_TYPE_COLORS)

**Single source of truth.** All sample-type color assignments across the entire app — global donut chart, slice donut chart, sampler type chart, PG timeline legend (inline HTML), and PG cadence heatmap fallback — MUST source from this map, **keyed by category NAME** (not array index).

```javascript
const SAMPLE_TYPE_COLORS = {
  'Air':     '#0072B2',  // Okabe-Ito blue         → --color-type-air
  'Plant':   '#009E73',  // Okabe-Ito teal-green    → --color-type-plant
  'Soil':    '#E69F00',  // Okabe-Ito amber         → --color-type-soil
  'Liquid':  '#56B4E9',  // Okabe-Ito sky blue      → --color-type-liquid
  'Unknown': '#999999',  // neutral gray             → --color-type-unknown
};
```

**Colorblind safety:** Okabe-Ito is the de-facto standard colorblind-safe categorical palette for scientific data. These five values are perceptually distinct under deuteranopia (affects ~8% of male readers) and tritanopia.

**Brand teal (`#0c5454` / `#0c9cb4`) is NOT a member of this data palette.** Per the Constitution's "Brand ≠ Data" rule, brand colors must never be used for sample-type data encoding. Note: Okabe Plant `#009E73` is hue-adjacent to bright teal `#0c9cb4` — they must not be used interchangeably.

**Legacy encodings retired by v2 (FE replaces with keyed lookups into SAMPLE_TYPE_COLORS):**
- `CHART_COLORS.sampleTypes` array → retired; use `SAMPLE_TYPE_COLORS` by key
- `CHART_COLORS.sliceSampleTypes` array → retired; use `SAMPLE_TYPE_COLORS` by key
- `CHART_COLORS.samplerType` array → retired; use `SAMPLE_TYPE_COLORS` by key
- `PG_TYPE_FILL` object → rebuilt to source hexes from `SAMPLE_TYPE_COLORS`
- `PG_TYPE_COLOR` object → rebuilt to source colors from `SAMPLE_TYPE_COLORS` (inline hex styles, no Tailwind color classes — the old bg-sky/emerald/amber/cyan classes are retired)
- Static index.html legend (~lines 300–303) → inline hex styles sourced from `SAMPLE_TYPE_COLORS`

## Pipeline Stage Colors

Three-stage pipeline bar chart (Collected → DNA Extracted → Sequenced). v1 palette used a CSU-green ramp; the Sequenced bar (`#4ade80`, ~1.7:1 on white) was effectively invisible. v2 replaces with a navy-to-teal sequential set where every step meets ≥3:1 non-text contrast on white (WCAG 1.4.11).

| Stage | Token | Hex | Contrast vs. White | WCAG (non-text 3:1) |
|-------|-------|-----|--------------------|----------------------|
| Collected | `--color-pipeline-collected` | `#1e3a5f` | ~11.1:1 | PASS (AAA) |
| DNA Extracted | `--color-pipeline-extracted` | `#2b6c8a` | ~5.6:1 | PASS (AA) |
| Sequenced | `--color-pipeline-sequenced` | `#4db6c4` | ~3.2:1 | PASS (non-text ≥3:1) |

**Replaces (all three steps changed):**
- pipeline[0] CSU green (Collected, v1) → `#1e3a5f` (`--color-pipeline-collected`)
- pipeline[1] green-600 (DNA Extracted, v1) → `#2b6c8a` (`--color-pipeline-extracted`)
- pipeline[2] green-400 / `#4ade80` (Sequenced, v1, ~1.7:1 FAIL) → `#4db6c4` (`--color-pipeline-sequenced`)

**Rationale:** The navy-to-teal progression is distinct from the Okabe sample-type palette, maintains clear sequential legibility, and eliminates the invisible Sequenced bar. `#4db6c4` at 3.2:1 passes non-text contrast (WCAG 1.4.11) for chart bars but MUST NOT be used as text.

**slicePipeline (Slice Panel):** `slicePipeline[0]` maps the same Collected stage and migrates to `#1e3a5f`. The other slice-pipeline steps (`#0f766e` Extracted, `#6d28d9` Sequenced) are unchanged.

## WCAG Contrast Notes

### Teal Text Restriction (A11Y — binding on FE, verified by AUD)

| Color | Hex | Contrast vs. White | WCAG Normal Text (4.5:1) |
|-------|-----|--------------------|--------------------------|
| Deep teal | `#0c5454` | ~9.1:1 | PASS AA + AAA |
| Bright teal | `#0c9cb4` | ~3.0:1 | **FAIL** (normal text) |

**Rule:** Bright teal (`#0c9cb4`) FAILS normal-text WCAG AA (~3:1 on white). It may ONLY be used for:
- Underlines and decorative borders (`border-b-2`, `border-l-4`)
- Active-accent marks (e.g. nav active underline)
- Large text (≥18px regular or ≥14px bold)
- Non-text contrast elements (icons, swatch halos, decorative)
- Elements on dark backgrounds where its contrast inverts favorably

**Normal-size link and active TEXT on white MUST use deep teal `#0c5454` (~9.1:1, AA + AAA).** Example: nav active link uses `color: #0c5454` (text) + `border-bottom: 2px solid #0c9cb4` (bright-teal underline accent).

This constraint is **binding on FE** (UI-002 sub-deliverable H) and is verified by AUD-001.

## Design Integrity Notes

- **Teal rebrand (v2):** BROADN teal replaces CSU green as the brand anchor. All former CSU-green brand uses (bg-green-900/800/100, text-green-800, border-green-800) migrate per the Migration Table above. See the full v1→v2 migration table for FE implementation guidance.
- **Orange filter envelope:** the current `filter-active-envelope` class applies a full-area background tint when a slice filter is active. Studio Clarity replaces this with a labeled chip in the section header — less visually intrusive, more informative (shows which filter is active).
- **Collapsible sidebar:** the slice sidebar is currently always-visible at `w-64` on desktop or hidden on mobile. Studio Clarity adds a collapse toggle. The collapsed state frees ~256px of horizontal space for charts on smaller desktops.
- **Inter font:** loaded via Google Fonts CDN. `body` in styles.css applies it correctly. Chart.js requires `Chart.defaults.font.family` to be set explicitly (app.js, before first `new Chart()`) — it does not inherit from CSS.
- **Scroll-linked nav:** nav active state tracks scroll position via JS. Ensure the `scroll-mt-20` offset on each `<section>` accounts for the `h-16` sticky nav height.
- **Dormant PG timeline legend:** the concurrent-timeline and project_group views have no live data currently. Migration of their color encodings (PG_TYPE_COLOR, PG_TYPE_FILL, static legend) is verified by grep only — screenshot verification is not possible.
