### FEATURE 5 — Rich Hover Tooltips Across All Charts (NORMAL)

**Problem:** Tooltip content is sparse or uniform across charts.

**Desired behavior — chart-by-chart:**

| Chart | Current tooltip | Desired tooltip |
|-------|----------------|-----------------|
| Global sample types donut | Project list | Pipeline stage breakdown for that type (how many collected/extracted/sequenced) |
| Global pipeline bar | Sample count | Sample type breakdown for that stage |
| Samples by site bar | Count | Top 3 sample types at that site + date range |
| Temporal line/bar | Count | Breakdown by sample type for that month |
| Map markers | Site name + count | Top sample types + collection date range |
| Slice types donut | % | Pipeline breakdown for that type (within the selected group) |
| Slice pipeline bar | Count | Sample type breakdown for that stage (within the selected group) |
| Slice temporal | Count | Sample type breakdown for that month (within the selected group) |
| Sampler type donuts | % | Which projects/locations use this sampler |

**Blocker for global charts:** `preprocess_data.py` needs cross-tab data for global-level type×pipeline and site×type breakdowns. BE task required.
**Slice charts:** Some cross-tab data exists in `tag_charts` — may partially satisfy slice tooltip needs. Verify before adding new BE data.

**Scope:** BE (new cross-tab keys in data.json) + FE (tooltip callback rewire per chart).

### FEATURE 6 — Project Banner (NORMAL)

**Problem:** When a project is selected from the sidebar, there's no immediate visual context about what that project is.

**Desired behavior:**
- When a project is selected, show a small banner at the top of the slice panel (above the charts)
- Banner contains: project name (heading), a short description blurb, and a thumbnail image
- For now: fill description with "Looking for feedback on this — [lorem ipsum filler]" and use a placeholder image (gray box or generic placeholder URL)
- Design should leave clear space/slots for real content to be dropped in later
- Only applies to the Project slice view (not Location or Lab Group)

**Scope:** FE-only initially (static placeholder content). Future: may need a lookup table of project descriptions — could be a small JSON sidecar file.

---

### FEATURE 7 — Active Tag Filter Banner (NORMAL)

**Problem:** When tags are selected and charts are filtered, there's no prominent indicator telling the user what filter is active.

**Desired behavior:**
- When one or more tag badges are active, show a banner in the slice panel above the charts: e.g. **"Filtered by: AM · Q1"** (one pill per active tag)
- Banner is dismissible (clicking X clears all tags and resets charts)
- Banner disappears when no tags are active
- Style: consistent with existing orange accent — orange border/background, not a full-bleed alert

**Scope:** FE-only. Integrates with existing `filterState.tags` and `applyFilter()`.
