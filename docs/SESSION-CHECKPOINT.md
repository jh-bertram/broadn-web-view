## Next Sprint Feature Plan

These are organized by feature area. Do not implement until the human confirms scope and priority for each sprint. Use `dispatch-task` for all multi-domain work.

---

### FEATURE 1 — "All Samples" Sidebar Toggle (HIGH PRIORITY)

**Problem:** Having both the global charts and the slice panel visible simultaneously is confusing. The user doesn't know which set of charts is "active."

**Desired behavior:**
- Add a fourth sidebar option: **"All BROADN Samples"** as the default/first selection
- Exactly one of the four options must be selected at all times (All Samples, Project, Location, Lab Group) — no null state
- When "All BROADN Samples" is selected: show the global charts (temporal, donut, pipeline, map, by-site bar), hide the slice panel
- When Project / Location / Lab Group is selected: show only that slice panel's charts, hide the global charts
- This makes the sidebar a true view switcher, not a supplemental filter

**Scope:** FE-only. No BE changes needed. State machine change in `renderView()`.

---

### FEATURE 2 — Chart Styling Cleanup (HIGH PRIORITY)

**Problem:** Charts have gray outlines/borders that feel heavy and unpolished.

**Desired behavior:**
- Remove the gray border/outline from all chart cards/containers
- Provide clearly commented locations in the HTML for every color and border value so the human can manually tune them (e.g., `/* CHART CARD BORDER — edit here */`)
- This applies to both global charts and slice panel charts

**Scope:** FE-only (CSS/Tailwind class changes + comments). No BE changes.

---

### FEATURE 3 — Time Distribution: Bar Charts (NORMAL)

**Problem:** Time distribution charts currently render as line charts, which implies continuity. Bar charts are more appropriate for discrete time period bins.

**Desired behavior:**
- Change all temporal/time-distribution charts from line to bar chart type
- Applies to: global temporal chart, all 3 slice temporal charts (`sliceProjectTemporalChart`, `sliceLocationTemporalChart`, `sliceLabGroupTemporalChart`), and the location time-of-day polar chart (convert to bar)
- Keep the "Mar '20" month label format and autoSkip disabled

**Scope:** FE-only. `buildTemporalChartOptions()` and inline chart constructors.

---

### FEATURE 4 — Samples by Site: Show All Locations (NORMAL)

**Problem:** The "Samples by Site" horizontal bar chart truncates or skips locations when there are many.

**Desired behavior:**
- All locations must be visible on the y-axis — no skipping, no truncation, no cap
- Chart height should expand dynamically to fit all entries rather than cropping
- If needed, add a scroll container for the chart area

**Scope:** FE-only. Chart options in `renderBySiteChart()` or equivalent.

---

### FEATURE 5 — Rich Hover Tooltips Across All Charts (NORMAL)

**Problem:** Tooltip content is sparse or uniform across charts. Many metadata columns exist that could provide useful context.

**Desired behavior — chart-by-chart:**

| Chart | Current tooltip | Desired tooltip |
|-------|----------------|-----------------|
| Global sample types donut | Project list | Pipeline stage breakdown for that type (how many collected/extracted/sequenced) |
| Global pipeline bar | Sample count | Sample type breakdown for that stage |
| Samples by site bar | Count | Top 3 sample types at that site + date range |
| Temporal line/bar | Count | Breakdown by sample type for that month |
| Map markers | Site name + count | Top sample types + collection date range |
| Slice types donut | % | Pipeline breakdown for that type (within the selected group) |
| Slice pipeline bar | Count | Sample type breakdown for that stage (within the selected group) |
| Slice temporal | Count | Sample type breakdown for that month (within the selected group) |
| Sampler type donuts | % | Which projects/locations use this sampler |

**Blocker for global charts:** `preprocess_data.py` needs cross-tab data for global-level type×pipeline and site×type breakdowns. BE task required.
**Slice charts:** Some cross-tab data exists in `tag_charts` — may partially satisfy slice tooltip needs. Verify before adding new BE data.

**Scope:** BE (new cross-tab keys in data.json) + FE (tooltip callback rewire per chart).

---

### FEATURE 6 — Project Banner (NORMAL)

**Problem:** When a project is selected from the sidebar, there's no immediate visual context about what that project is.

**Desired behavior:**
- When a project is selected, show a small banner at the top of the slice panel (above the charts)
- Banner contains: project name (heading), a short description blurb, and a thumbnail image
- For now: fill description with "Looking for feedback on this — [lorem ipsum filler]" and use a placeholder image (gray box or generic placeholder URL)
- Design should leave clear space/slots for real content to be dropped in later
- Only applies to the Project slice view (not Location or Lab Group)

**Scope:** FE-only initially (static placeholder content). Future: may need a lookup table of project descriptions — could be a small JSON sidecar file.

---

### FEATURE 7 — Active Tag Filter Banner (NORMAL)

**Problem:** When tags are selected and charts are filtered, there's no prominent indicator telling the user what filter is active. The amber "no match" notice exists for zero-match edge cases, but no general "filtered by X" state banner.

**Desired behavior:**
- When one or more tag badges are active, show a banner in the slice panel above the charts: e.g. **"Filtered by: AM · Q1"** (one pill per active tag)
- Banner is dismissible (clicking X clears all tags and resets charts)
- Banner disappears when no tags are active
- Style: consistent with existing orange accent — orange border/background, not a full-bleed alert

**Scope:** FE-only. Integrates with existing `filterState.tags` and `applyFilter()`.
